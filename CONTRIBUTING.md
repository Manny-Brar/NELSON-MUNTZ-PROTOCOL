## How To Contribute

### 1. Found a Bug? üî¥

Open an issue. Include:
- What you were trying to do
- What happened instead
- Error messages (the whole thing, not just "it didn't work")
- Your setup (OS, Claude Code version)

**Don't:** Open an issue that just says "it's broken." That's lazy. I don't respect lazy.

### 2. Got an Idea? üí°

Open an issue first. Describe:
- What you want to add
- Why it's useful
- How it fits the Nelson philosophy (fresh context, single focus, etc.)

**Don't:** Just submit a PR without talking first. I might not want your "improvement."

### 3. Want to Submit Code? ü•ä

1. Fork the repo
2. Create a branch (`git checkout -b feature/your-thing`)
3. Make your changes
4. Test it (I will check)
5. Submit a PR with a clear description

## Code Standards

### Keep It Simple

- No over-engineering
- If it works, don't make it "elegant"
- Comments only where actually needed

### Follow the Patterns

- Look at existing code
- Match the style
- Don't reinvent what's already there

### Test Your Stuff

- Run the scripts
- Make sure nothing breaks
- If you add features, show me they work
